# Logos Panel Policy

Use Logos when extra certainty is worth the latency:

- major refactors
- architecture changes
- security or destructive operations
- model/router decisions
- assistant behavior changes
- production or data-loss risk
- user explicitly asks for Logos, three opinions, GPT-Fusion, or Fable-like synthesis

Do not use Logos for:

- ordinary chat
- simple summaries
- small edits
- one-off commands
- tasks where the answer can be verified faster than a panel can reason

The third opinion is not a vote. It is a defect-finding surface.

If multiple providers are unavailable, use independent multi-pass review and explicitly say it was not multi-model.
